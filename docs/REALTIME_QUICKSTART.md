## 🚀 Как это работает

### Асинхронные соревнования (уже работает)

```
1. Учитель создаёт соревнование
2. Ученики проходят тренажёр в течение 24 часов
3. Результаты сохраняются в IndexedDB
4. Лидерборд обновляется при загрузке
```

**Статус:** ✅ Работает в текущей версии

### Синхронные (Live) соревнования (требует backend)

```
1. Учитель создаёт соревнование на 15:00
2. В 14:55 ученики получают уведомление
3. В 15:00 все начинают одновременно
4. Счёт обновляется каждые 1-2 секунды
5. Лидерборд обновляется каждые 5 секунд
6. В 15:05 финиш и награждение
```

**Статус:** ⚠️ Требуется backend сервер

## 📋 Что нужно для запуска Live-соревнований

### Минимальные требования

1. **Backend Server** (Node.js)
   - WebSocket для real-time связи
   - REST API для CRUD операций
   - JWT авторизация

2. **Redis** (опционально, но рекомендуется)
   - Кэш лидербордов
   - Pub/Sub для событий

3. **PostgreSQL** (опционально)
   - Хранение результатов
   - История соревнований

### Варианты реализации

#### Вариант 1: Firebase (Быстрый старт) ⭐ Рекомендую для начала

**Плюсы:**
- Не нужен свой сервер
- Realtime Database из коробки
- Бесплатно до 50K прочтений/день

**Как использовать:**
```typescript
import { getDatabase, ref, set, onValue } from 'firebase/database';

const db = getDatabase();

// Отправка счёта
const scoreRef = ref(db, `competitions/${id}/scores/${userId}`);
set(scoreRef, { score, accuracy, timestamp: Date.now() });

// Подписка на лидерборд
const lbRef = ref(db, `competitions/${id}/leaderboard`);
onValue(lbRef, (snapshot) => {
  const data = snapshot.val();
  updateUI(data);
});
```

**Время интеграции:** 1-2 дня

#### Вариант 2: Свой WebSocket сервер (Полный контроль)

**Плюсы:**
- Полный контроль над логикой
- Нет ограничений по запросам
- Дешевле при большой нагрузке

**Минусы:**
- Нужен VPS (~$20/мес)
- Сложнее в настройке
- Нужно поддерживать

**Время интеграции:** 2-3 недели

Код сервера уже создан в папке `backend/`

## 🔧 Технические детали

### WebSocket Протокол

**Клиент → Сервер:**
```json
{
  "type": "competition:score_update",
  "competitionId": "comp123",
  "score": 150,
  "accuracy": 0.95,
  "timestamp": 1711551600000
}
```

**Сервер → Клиент:**
```json
{
  "type": "leaderboard:update",
  "competitionId": "comp123",
  "leaderboard": [
    {"rank": 1, "userId": "user1", "score": 200},
    {"rank": 2, "userId": "user2", "score": 180}
  ],
  "userRank": 2,
  "timestamp": 1711551600000
}
```

### Anti-Cheat

- Проверка скорости реакции (> 100ms)
- Сравнение с историей пользователя
- Rate limiting (10 сообщений/сек)
- Серверная валидация
