## ⚔️ Этап 2: Соревновательная база (Недели 3-4)

### 2.1. Система вызовов (PvP)
**Компоненты:**
- `src/features/competitions/components/ChallengeModal.tsx`
- `src/features/competitions/components/ChallengeList.tsx`
- `src/features/competitions/components/ChallengeCard.tsx`

**Механика:**
```
1. Ученик выбирает соперника (из класса или другого класса)
2. Выбирает тренажёр и длительность
3. Отправляет вызов
4. Соперник получает уведомление
5. Принимает/отклоняет
6. Оба проходят тренажёр в течение 24 часов
7. Система определяет победителя по XP/точности
8. Обновляется рейтинг
```

**API:**
```typescript
// Repository
competitionRepository.createChallenge(challenge)
competitionRepository.respondToChallenge(id, accept)
competitionRepository.completeChallenge(id, score)
competitionRepository.listChallenges(userId)

// Хуки
useCompetitionChallenges(userId)
useSendChallenge()
useRespondToChallenge()
```

**Фильтры:**

- Навык: attention / memory / reaction / math / logic

```
┌─────────────────────────────────────────────────────┐

│  Offline режим (локальные сессии)                  │

└─────────────────────────────────────────────────────┘
```

### 3.2. Real-time механика
**Компоненты:**
- `src/features/competitions/components/LiveCompetition.tsx`
- `src/features/competitions/components/LiveLeaderboard.tsx`
- `src/features/competitions/components/CompetitionTimer.tsx`

**Сценарий:**
```
1. Учитель создаёт соревнование
   - Название: "Математический бой"
   - Режим: Sprint Math
   - Время: 15:00, 5 минут
   - Участники: 11"А" vs 11"Б"

2. Ученики получают уведомление за 5 минут

3. В 15:00 старт
   - Все начинают одновременно
   - Таймер на экране
   - Лидерборд обновляется каждые 10 сек

4. В 15:05 финиш
   - Подсчёт результатов
   - Награждение (XP, бейджи)
   - Сохранение в историю
```

### 3.3. Командные соревнования
**Механика:**
```
Класс vs Класс:
- Сумма XP всех участников
- Средняя точность
- Количество завершивших

Победитель определяется по:
1. Суммарный XP (60%)
2. Средняя точность (30%)
3. Активность (10%)
```

## 🔧 Технические требования

### Производительность
- Лидерборд: < 100ms обновление
- Heatmap: < 50ms рендер
- Real-time: < 1s задержка

### Offline-first
- Все данные кэшируются в IndexedDB
- Синхронизация при подключении
- Конфликты: server wins

### Безопасность
- Валидация результатов на сервере
- Anti-cheat (аномалии реакции)
- Rate limiting для API

## 📊 Метрики успеха

### Для учеников:
- % активных за неделю (>60%)
- Среднее число сессий на ученика (>3)
- Участие в соревнованиях (>40%)

### Для учителей:
- Время на создание соревнования (<2 мин)
- Просмотр статистики класса (1 раз в неделю)
- Удовлетворённость (опрос >4/5)
