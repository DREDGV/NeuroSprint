# Деплой `нейроспринт.рф`

## Что у вас сейчас

- Домен `нейроспринт.рф` зарегистрирован.
- DNS обслуживается через `ns10.hoster.ru` / `ns11.hoster.ru`.
- Сам по себе домен сайт не хранит. Нужен хостинг, куда будет выкладываться сборка проекта.

## Что лучше для текущего NeuroSprint

Текущий фронтенд можно выкладывать как **статический сайт**:

- сборка через `Vite`
- интерфейс на `React`
- пользовательские данные хранятся локально через `Dexie` / `IndexedDB`
- для обычной работы основной части приложения отдельный сервер не нужен

Это значит, что проще всего развернуть фронтенд на **Vercel** и просто привязать туда домен.

Отдельный `backend/` в репозитории сейчас нужен только для real-time соревнований. Его лучше выносить позже на VPS и публиковать на отдельном поддомене, например:

- `api.нейроспринт.рф`
- `ws.нейроспринт.рф`

## Что уже подготовлено в репозитории

В корне проекта добавлен `vercel.json`:

- Vercel понимает, что это `Vite`
- output directory = `dist`
- все SPA-маршруты вроде `/training/...` переписываются на `index.html`

Это важно, иначе прямые переходы по адресам внутри приложения будут отдавать 404.

## Пошаговый вариант запуска

### 1. Залить проект в GitHub

Если репозиторий ещё не в GitHub:

1. Создайте новый приватный или публичный репозиторий.
2. Запушьте туда текущий проект.

### 2. Подключить проект в Vercel

1. Зайдите в Vercel.
2. Нажмите `Add New Project`.
3. Импортируйте репозиторий NeuroSprint.
4. Важно: Vercel возьмёт код из GitHub, а не текущие локальные незакоммиченные файлы.
5. Проверьте настройки сборки:
   - Framework Preset: `Vite`
   - Build Command: `npm run build`
   - Output Directory: `dist`
6. Запустите первый deploy.

### 3. Привязать домен в Vercel

В настройках проекта Vercel:

1. Откройте `Settings -> Domains`.
2. Добавьте:
   - `нейроспринт.рф`
   - `www.нейроспринт.рф`
3. Если какая-то панель не принимает кириллицу, используйте punycode-форму домена, которую у вас уже показывает регистратор в скобках.

### 4. Настроить DNS у регистратора

В вашей панели `hoster.ru` сейчас уже есть корневая A-запись:

- `A -> 31.28.24.249`

Её нужно заменить под Vercel.

Сделайте так:

#### Корень домена

- Имя: оставить пустым
- Тип: `A`
- Значение: `76.76.21.21`
- TTL: `1800`

То есть:

1. удаляете старую A-запись `31.28.24.249`
2. создаёте новую A-запись на `76.76.21.21`

#### Поддомен `www`

- Имя: `www`
- Тип: `CNAME`
- Значение: возьмите то, которое покажет сам Vercel в `Settings -> Domains`
- TTL: `1800`

Обычно Vercel использует значение вида `cname.vercel-dns-0.com`, но по их документации у проекта может быть и своё точное значение, поэтому в панели регистратора лучше ориентироваться на подсказку Vercel для вашего домена.

MX-запись для почты можно оставить как есть, если планируете использовать почту на домене.

## Что получится после этого

- `https://нейроспринт.рф` будет открывать фронтенд
- `https://www.нейроспринт.рф` можно в Vercel направить на основной домен
- HTTPS-сертификат Vercel выпустит автоматически после того, как DNS обновится

## Сколько ждать после DNS

Обычно:

- первые изменения видны через 5-30 минут
- иногда полное обновление занимает до 24 часов

## Если захотите публиковать backend позже

Тогда схема будет такой:

- фронтенд: `нейроспринт.рф` на Vercel
- backend API: `api.нейроспринт.рф` на VPS
- websocket: `ws.нейроспринт.рф` на VPS

Тогда в DNS добавляются ещё записи:

- `api` -> `A` -> IP вашего VPS
- `ws` -> `A` -> IP вашего VPS

Но для первого запуска NeuroSprint это не обязательно.

## Проверка перед релизом

Локально:

```powershell
npm run build
```

Если сборка проходит, фронтенд готов к статическому деплою.
