# NeuroSprint Unified Plan (RU)

Обновлено: 2026-03-10  
Статус документа: канонический рабочий roadmap

## 1. Назначение документа
Это главный рабочий документ проекта.

Только он должен использоваться как основной источник решений по:
- текущему продуктовому курсу;
- фактическому статусу модулей;
- очередности ближайших продуктовых шагов;
- статусу `Progress System`;
- месту `Training Hub` и experimental-линии в продукте.

Если другие старые roadmap/spec документы противоречат этому файлу, приоритет у этого документа.

## 2. Фактическое состояние проекта
### Официальные production-ready модули
- Таблица Шульте
- Sprint Math
- Reaction
- N-Back Lite
- Decision Rush
- Memory Grid

### Production-ready инфраструктура прогресса
- `Daily Training`
- `Levels + Achievements`
- XP / rewards pipeline при сохранении сессии
- level / achievement widgets и feedback-компоненты в основном UI

### Частично интегрированные, но ещё не завершённые продуктовые слои
- `Skill Guidance` и ранний `Skill Roadmap`
- `Training Hub` как skill-based экран выбора
- `Pattern Recognition` как рабочий модуль вне главного текущего фокуса

### Alpha / prototype-линия
- `Block Pattern Recall` — alpha-прототип
- `Memory Match` и `Spatial Memory` уже переведены в официальный контур памяти

## 3. Неподвижные правила
1. Не переводить experimental-модуль в официальный контур без полного пользовательского цикла `route -> setup -> session -> save -> stable UI`.
2. Не включать alpha-модули в официальный `stats / recommendation / challenge / progress`, пока нет отдельного решения о promotion.
3. Не открывать новый большой продуктовый трек, пока не стабилизирован текущий активный кандидат.
4. После изменения фактического статуса проекта обновлять только:
   - `docs/NeuroSprint_UNIFIED_PLAN_RU.md`
   - `docs/NeuroSprint_Execution_Status_RU.md`
   - `docs/CHANGELOG_RU.md`
5. Все старые roadmap/spec файлы считать reference/архивом, а не рабочими инструкциями.

## 4. Зафиксированное решение по новым memory-модулям
- `Memory Match` — основной memory-модуль.
- `Spatial Memory` — основной модуль на пространственную память.
- `Block Pattern Recall` — остаётся alpha-прототипом.

Причина:
- `Memory Match` и `Spatial Memory` уже закрывают разные сценарии памяти без дублирования механики.
- `Block Pattern Recall` пока не должен распылять фокус команды до завершения product-pass.

## 6. Частично начатые, но ещё не доведённые этапы
### Progress System / Phase 3 — Skill Map + Strength/Weakness Analysis
Этап уже начат, но ещё не доведён до стабильного продукта.

Что уже есть:
- `skillProfile`;
- `skillGuidance`;
- `skillRoadmap`;
- ранняя интеграция в `HomePage`, `StatsPage`, `PreSessionPage`, `TrainingHubPage`, `DailyTrainingWidget` и challenge/recommendation-слои.

Что ещё не завершено:
- единый и чистый продуктовый язык;
- полная стабильность и согласованность UI;
- доведение до канонического skill map experience;
- финальная quality pass по текстам, кодировке и пользовательской ясности.

### Training Discovery & Feedback
Этот трек уже начат частично, но ещё не считается завершённым.

Что уже есть:
- `Training Hub` как skill-based экран выбора;
- группировка тренажёров по навыкам;
- отдельный блок experimental trainers;
- верхний сценарий выбора “что тренировать сейчас”.

Что ещё не завершено:
- финальная visual hierarchy;
- trainer feedback после сессии;
- внутренняя аналитика качества тренажёров;
- окончательная развязка между official и experimental UX-слоями.

## 7. Текущий активный фокус
### Stream A — Production-pass главного каталога памяти
Цель:
- удержать `Memory Match` и `Spatial Memory` как сильные основные модули без деградации UX.

Что входит:
- onboarding / правила;
- игровой цикл;
- кликабельность и целостность поля;
- визуальный polish;
- результат сессии;
- безопасное сохранение и корректный session flow.

Что не входит:
- параллельное доведение `Block Pattern Recall`.

### Stream B — Доведение `Training Hub`
Цель:
- сделать экран тренажёров понятным как skill-based навигатор, а не как сырой каталог карточек.

Что входит:
- skill-first структура;
- понятный верхний сценарий выбора;
- visual hierarchy между featured и secondary trainers;
- отдельный и честный experimental block.

Что не входит:
- публичный рейтинг тренажёров;
- большой marketplace-style каталог;
- расползание на все product pages сразу.

### Stream C — Stabilization текущих продуктовых слоёв
Цель:
- сократить расхождение между уже внедрёнными progress/discovery слоями и их фактическим качеством.

Что входит:
- build/test стабильность;
- чистка битых строк и кодировок;
- устранение противоречий между roadmap и реальным состоянием кода;
- доводка раннего skill guidance/roadmap до чистого состояния.

## 8. Следующие этапы после текущего фокуса

### Next Phase 3 — Skill Map + Strength & Weakness Analysis (с заделом на PvP)

**Цель:** Создать понятную карту навыков с рекомендациями и конкурентным сравнением.

**Открывать только после:**
- `Memory Match` и `Spatial Memory` стабилизированы как основные модули
- `Training Hub` доведён до рабочего UX
- Ранние skill/guidance слои очищены и согласованы

**Что включает:**
- Процентили навыков (симулированные "виртуальные игроки")
- 9 рекомендаций (по 3 на уровень: weak/medium/strong)
- 10 достижений за навыки (50/80 очков)
- Вкладка "🎯 Навыки" в StatsPage
- **Архитектура для будущего PvP:**
  - Абстракция benchmarks (virtual → server)
  - Поля для лиг/рейтингов в БД
  - Скрытые PvP достижения

**Что НЕ включает:**
- Отдельные уровни для навыков (Phase 4)
- Реальные лидерборды (нужен бэкенд)
- Древовидные структуры навыков

**Критерии завершения:**
- [ ] Пользователь видит процентили по каждому навыку
- [ ] Показываются ТОП-3 рекомендации
- [ ] 10 новых достижений за навыки
- [ ] 20+ unit тестов
- [ ] Build без ошибок
- [ ] CHANGELOG обновлён

**Оценка:** 35-45 часов

---

### Next Phase 4 — AI Recommendations + PvP Foundation

**Цель:** Добавить персональный guidance-слой и подготовить инфраструктуру для онлайн-соревнований.

**Открывать только после:**
- Phase 3 завершена
- Есть стабильные данные о прогрессе пользователей
- memory-модули уже стабильно живут в official-контуре

**Что включает:**
- **Лиги:** Бронза → Серебро → Золото → Алмаз (10 лиг как Duolingo)
- **Рейтинги:** Еженедельные соревнования (виртуальные → реальные)
- **Недельные квесты:** Расширение Daily Challenge
- **AI Recommendations:** Персонализация на основе истории
- **Приватные лидерборды:** Соревнование с друзьями/классами

**Архитектурные решения:**
- Лиги: Интерфейс готов в Phase 3, скрывается до релиза
- Рейтинги: Абстракция benchmarks для лёгкой миграции на сервер
- Квесты: Расширение existing dailyChallengeRepository

**Критерии завершения:**
- [ ] 10 лиг с повышением/понижением
- [ ] Еженедельные квесты с наградами
- [ ] AI-рекомендации (персонализированные)
- [ ] Приватные лидерборды (для классов)
- [ ] 30+ unit/integration тестов

**Оценка:** 50-70 часов

---

### Next Phase 5 — Mass Release Preparation

**Цель:** Подготовка к массовому релизу с онлайн-соревнованиями.

**Что включает:**
- **Бэкенд:** Сервер для лидербордов, аутентификация, синхронизация
- **PvP режимы:** Reaction Duel, Memory Battle, Attention Race
- **Виртуальная валюта:** NeuroCoins за достижения
- **Кастомизация:** Аватары, темы за валюту
- **Заморозка streak:** Защита от пропуска (за валюту)
- **Мобильные приложения:** iOS/Android (PWA → native)

**Критерии завершения:**
- [ ] Сервер работает стабильно
- [ ] PvP режимы тестируются
- [ ] 1000+ beta пользователей
- [ ] Мобильные приложения в магазинах

**Оценка:** 200-300 часов

## 10. Experimental Track
### Experimental Trainers
- `Block Pattern Recall` — alpha.

Правило:
- experimental-линия должна быть видимой пользователю честно, но не смешиваться с official progress-системой на одном уровне продукта.

## 10. Что не делать сейчас
- Не открывать параллельно новый большой gameplay-трек.
- Не возвращать `Spatial Memory` и `Memory Match` в experimental-статус без серьёзного продуктового основания.
- Не размазывать усилия между `Training Hub`, `Skill Map`, `AI Recommendations` и новыми gameplay-ветками одновременно.
- Не считать ранний `skillRoadmap` уже завершённой полноценной skill-map системой.

## 11. Следующие конкретные шаги
1. Довести верхний контур `Training Hub` до действительно сильного skill-based выбора.
2. Провести stabilization pass по уже внедрённым progress/discovery слоям.
3. Вернуться к `Block Pattern Recall` только после закрытия текущего продуктового контура.
4. Только после этого открывать полноценный product-pass по `Skill Map + Strength/Weakness Analysis`.
5. Затем переходить к `AI Recommendations`.
