# NeuroSprint — План работы для AI-агента

Обновлено: 2026-03-07
Текущая версия кода: `0.5.0-dev.5`

## 1. Цель
Работать поэтапно, без хаоса и без расширения проекта во все стороны сразу.

## 2. Текущее состояние проекта
### Официальные модули
- Таблица Шульте
- Sprint Math
- Reaction
- N-Back Lite
- Decision Rush
- Memory Grid

### Частично интегрированные / не в приоритете
- Pattern Recognition

### Alpha-модули
- Memory Match
- Block Pattern Recall
- Spatial Memory

## 3. Зафиксированное решение по alpha-модулям
- `Memory Match` — следующий production-кандидат.
- `Block Pattern Recall` — alpha-прототип.
- `Spatial Memory` — hold / redesign.

## 4. Обязательные правила работы
1. Делать по одному значимому шагу за сессию.
2. Не расширять задачу за пределы согласованной области.
3. Не добавлять alpha-модуль в официальные `stats / recommendation / challenge`, пока не готовы:
   - доменные типы,
   - маршрутный поток,
   - сохранение сессии,
   - базовая статистика.
4. Не делать широкий рефакторинг без прямой пользы для пользователя.
5. После каждого этапа, который меняет фактический статус проекта, обновлять:
   - roadmap,
   - execution status,
   - changelog.

## 5. Ближайший рабочий порядок
1. Синхронизация docs — завершена.
2. Следующая основная реализационная сессия — `Memory Match`.
3. После неё — точечная оценка, нужен ли следующий шаг по `Block Pattern Recall`.
4. `Spatial Memory` не реализовывать дальше без отдельного продуктового решения.

## 6. Чеклист перед началом любой сессии
- Понять точную область работ.
- Прочитать только нужные файлы.
- Не трогать соседние модули без необходимости.
- Держать изменения минимальными и обратимо-понятными.

## 7. Чеклист после завершения сессии
- Проверить сборку и нужные точечные тесты.
- Зафиксировать, что реально изменилось.
- Обновить документы, если изменился фактический статус проекта.

## 8. Следующий конкретный шаг
- Начать реализационную сессию по `Memory Match` как следующему официальному модулю.
