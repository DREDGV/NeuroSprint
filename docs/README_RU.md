# NeuroSprint Docs Index (RU)

Обновлено: 2026-03-12

## Коротко: где мы сейчас
### Что делаем сейчас
- Удерживаем `Memory Match` и `Spatial Memory` как сильные основные модули памяти.
- Доводим `Training Hub` как skill-based экран выбора тренировки.
- Чистим и стабилизируем уже внедрённые progress/discovery слои.

### Что делаем потом
- Полноценный product-pass по `Skill Map + Strength/Weakness Analysis`.
- Затем отдельный слой `AI Recommendations`.

### Что не трогаем сейчас
- Не продвигаем `Block Pattern Recall` дальше alpha.
- `Spatial Memory` уже переведён в основной каталог; дальнейшие правки делаем как product-pass, а не как experimental-ветку.
- Не смешиваем alpha-модули с официальным progress-loop.
- Не открываем новый большой продуктовый трек параллельно `Memory Match` и `Training Hub`.

## Канонические рабочие документы
Использовать в повседневной работе только их:

1. `docs/NeuroSprint_UNIFIED_PLAN_RU.md`
   - главный roadmap и основной источник продуктовых решений;
2. `docs/NeuroSprint_Execution_Status_RU.md`
   - короткий фактический снимок проекта после параллельной работы нескольких AI-агентов;
3. `docs/CHANGELOG_RU.md`
   - история изменений по факту.

## Как ими пользоваться
- `UNIFIED_PLAN` — единственный канонический roadmap.
- `Execution_Status` — актуальное состояние кода и ближайшего фокуса.
- `CHANGELOG` — история уже внесённых изменений, а не план.

Если старые документы или архивные roadmap расходятся с этими файлами, опираться нужно на `UNIFIED_PLAN` и `Execution_Status`.
