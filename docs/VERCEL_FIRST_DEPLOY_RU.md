# Первый деплой NeuroSprint в Vercel

Эта инструкция рассчитана на ваш текущий этап:

- сайт публикуется только с индивидуальными тренировками
- `main` будет production-веткой
- новые функции пока остаются скрытыми через feature flags

## Что важно понять заранее

Vercel не берёт код прямо из вашей папки на компьютере.

Он берёт код из GitHub. Это значит:

1. локально вы можете что угодно менять и проверять
2. на сайт попадёт только то, что уже закоммичено и запушено в GitHub

Если нужные изменения ещё только локально, их надо сначала подготовить к коммиту и запушить.

## Что у проекта уже готово

В репозитории уже есть:

- `vercel.json`
- production feature flags для первого публичного релиза в `.env.example`
- SPA rewrite для внутренних маршрутов вроде `/training/...`

То есть отдельную ручную конфигурацию роутинга в Vercel для фронтенда делать не нужно.

## Что нажимать в Vercel

1. Откройте `https://vercel.com/dashboard`
2. Нажмите `Add New...`
3. Выберите `Project`
4. Найдите репозиторий `DREDGV/NeuroSprint`
5. Нажмите `Import`

Если репозиторий не видно:

- нажмите подключение GitHub
- дайте Vercel доступ к вашему GitHub
- если потребуется, откройте список репозиториев и явно разрешите `DREDGV/NeuroSprint`

## Что указывать в настройках проекта

На экране импорта проекта:

- `Project Name`: `neurosprint`
- `Framework Preset`: `Vite`
- `Root Directory`: оставить корень репозитория
- `Build Command`: `npm run build`
- `Output Directory`: `dist`

Если Vercel сам распознал эти значения правильно, ничего вручную менять не нужно.

## Какие env добавить на первый релиз

В блоке `Environment Variables` добавьте:

```env
VITE_FEATURE_CLASSES_UI=false
VITE_FEATURE_COMPETITIONS_UI=false
VITE_FEATURE_GROUP_STATS_UI=false
VITE_FEATURE_ONLINE_COMPETITIONS=false
```

Пока этого достаточно.

`VITE_API_BASE_URL` и `VITE_WS_URL` сейчас не нужны, потому что первый релиз будет без сетевых режимов.

## Что будет после нажатия Deploy

После `Deploy` Vercel создаст первый адрес вида:

- `https://neurosprint-....vercel.app`

Это ещё не ваш домен, а технический адрес проекта.

Сначала проверьте на нём:

- открывается главная
- открываются тренировки
- работает переход по внутренним маршрутам
- не видно скрытых разделов соревнований и классов

## Как привязать домен `нейроспринт.рф`

Когда первый деплой завершится:

1. Откройте проект в Vercel
2. Перейдите в `Settings -> Domains`
3. Добавьте:
   - `нейроспринт.рф`
   - `www.нейроспринт.рф`

Если какая-то форма не принимает кириллицу, используйте punycode-домен, который показывает ваш регистратор.

## Что менять в DNS у регистратора

У вас DNS остаётся на стороне `hoster.ru`. Nameserver'ы менять не нужно.

Нужно только поменять записи.

### Корневой домен

- удалите старую `A` запись на `31.28.24.249`
- создайте новую `A` запись:
  - имя: пусто или `@`
  - значение: `76.76.21.21`
  - TTL: `1800`

### Поддомен `www`

- создайте `CNAME` запись:
  - имя: `www`
  - значение: ровно то, что покажет Vercel в `Settings -> Domains`
  - TTL: `1800`

По официальной документации Vercel `76.76.21.21` и `cname.vercel-dns-0.com` являются общими значениями, но у конкретного проекта Vercel может показать своё точное значение для `CNAME`. Для `www` ориентируйтесь именно на значение из Vercel.

## Что делать дальше после первого запуска

После первого production-деплоя рабочий процесс будет таким:

1. новые функции делаете в ветке `feature/...`
2. пушите ветку в GitHub
3. Vercel сам создаёт preview deployment
4. проверяете новую функцию по preview URL
5. только потом мержите в `main`

Это и есть самый удобный путь для развития проекта без поломки основного сайта.
