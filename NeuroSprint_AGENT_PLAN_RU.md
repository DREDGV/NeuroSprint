# NeuroSprint — План работ для ИИ‑агента (поэтапно, без путаницы)
Версия: 1.0  
Цель документа: дать ИИ‑агенту понятный **пошаговый план** улучшения NeuroSprint с критериями готовности.

---

## 0) Правила работы (чтобы был прогресс, а не хаос)

**Общие правила**
1. Работать **итерациями**: один этап → проверка → коммит → следующий этап.
2. Не делать “рефакторинг ради рефакторинга”. Каждый шаг должен улучшать UX/функционал.
3. Не добавлять новые модули, пока не стабилизированы навигация/старт/статистика.

**Git‑процесс**
- Для каждого этапа создавать ветку: `feat/<short-name>`
- После этапа: `npm test` + `npm run build` (+ e2e, если есть) → коммит → PR/merge

**Команды проверки**
```bash
npm install
npm run dev
npm test
npm run build
# если есть Playwright:
npm run test:e2e
```

**Definition of Done для этапа**
- UX соответствует описанию этапа
- Нет критических багов
- Проходит сборка
- Проходит минимум unit/smoke тесты (если есть)
- Код не “ломает” другие режимы/игры

---

## 1) Этап 1 — Навигация и запуск игр (убрать путаницу)

### Цель
Сделать запуск любой игры одинаковым и понятным:
**Главная → Выбор игры → Настройки → ОДНА кнопка “Начать” → Игра → Результат**

### Задачи
1. Убрать “двойной старт” и дублирующие точки запуска.
2. Оставить **один основной путь** запуска:
   - `SetupPage` (настройки) → `SessionPage` (игра)
3. Если где-то есть “pre-session/план дня” — оставить **как дополнительную опцию**, но не основной старт.

### Критерии готовности
- Пользователь может начать Шульте (и Sprint Math, если есть) за 2–3 клика.
- Нет двух разных кнопок, которые стартуют одну и ту же сессию по-разному.
- Тесты (smoke/e2e) обновлены под новый UX.

### Результат этапа
Пользователь не путается. ИИ‑агент больше не “латаeт UX” в разных местах.

---

## 2) Этап 2 — Поле Шульте строго квадратное + удобство на телефоне

### Цель
Игровое поле Шульте всегда квадратное (на любых экранах), клетки удобны для нажатия.

### Задачи
1. Контейнер поля: `aspect-ratio: 1/1`, ограничение по ширине `min(90vw, 520px)`.
2. Клетки:
   - достаточный размер (ориентир: ≥ 48px высота/ширина)
   - `gap` адаптивный (через `clamp`)
   - убрать “слипание” на 6×6 (если есть такие режимы)
3. Тач‑оптимизация:
   - `touch-action: manipulation`
   - убрать highlight, если мешает

### Критерии готовности
- На телефоне и ноутбуке поле **не растягивается** и не становится прямоугольным.
- Цифры читаемые, кнопки не слишком мелкие.

---

## 3) Этап 3 — Современный визуал + дружелюбные темы (дети/взрослые)

### Цель
Сделать 3–4 темы, которые реально меняют стиль (не “чуть другой фон”).

### Минимальные темы
1. **Kids Bright** — яркая, дружелюбная, “мультяшная” (жирный округлый шрифт)
2. **School Light** — нейтральная, светлая
3. **Pro Dark** — строгая, взрослая
4. (опция) **Neon Focus** — контрастная

### Задачи
1. Ввести/упорядочить “токены темы”: фон, сетка, клетки, текст, акцент, шрифт, радиусы.
2. Для Kids:
   - округлый жирный шрифт цифр
   - больше радиусы
   - мягкие тени/обводки (аккуратно)
3. Сделать переключатель темы в понятном месте:
   - либо в `SetupPage`
   - либо в “Настройки профиля пользователя”

### Критерии готовности
- Видно, что это разные темы.
- Детская тема выглядит “дружелюбно”, а не как тех. демо.

---

## 4) Этап 4 — Простой экран результата + подсказки

### Цель
После каждой игры пользователь понимает: **как он сыграл** и **что улучшить**.

### Задачи (общие для всех игр)
1. Экран результата показывает:
   - итог (время/скорость/точность)
   - “лучший результат” и “сравнение с прошлым разом”
2. Встроить короткие подсказки:
   - “Ошибка снижает итог”
   - “Старайся держать точность выше 85%”
3. Добавить “Как играть” (tooltip/модал) на экране настроек.

### Критерии готовности
- После сессии есть понятный итог и рекомендация.
- Подсказки не мешают, но помогают.

---

## 5) Этап 5 — Статистика: просто, понятно, показывает прогресс

### Цель
Сделать статистику, которая “работает на мотивацию”:
**видно прогресс**, а не “линии от потолка”.

### Минимальный набор (MVP статистики)
1. Блок сверху:
   - “Ты стал быстрее на +X% за 14 дней” (или за 30)
2. Личный рекорд + дата
3. Один главный график:
   - линия “твой тренд” (лучшее/среднее)
   - сглаживание (скользящее среднее 7 дней)
4. Режим сравнения (по желанию):
   - показать “среднее по всем пользователям” или “медиану”

### Сравнение пользователей
На локальном уровне (без сервера):
- “Ты в топ‑3 из 10” (по выбранной игре/режиму за неделю)
- или “Твой процентиль: 70%”

### Критерии готовности
- Пользователь открывает /stats и за 10 секунд понимает прогресс.
- Есть сравнение с другими (хотя бы простое).

---

## 6) Этап 6 — Соревновательность (локальная) + лидерборды

### Цель
Дать “эффект соревнования” без сервера.

### Задачи
1. Экран “Лидерборд”:
   - выбрать игру/режим
   - выбрать период: “сегодня / неделя / месяц”
   - показать Top‑10
2. Подсветить текущего пользователя.
3. Ввести “Daily Challenge” (опционально):
   - ежедневная цель (например: Timed Шульте 60 сек → набери ≥ X)
   - отметка “выполнено/не выполнено”

### Критерии готовности
- Можно быстро увидеть кто лучший в группе пользователей на устройстве.
- В школе можно сравнивать учеников.

---

## 7) Этап 7 — Новые игры (2 модуля в ближайшем релизе)

> Важно: новые игры добавляем только после этапов 1–5 (иначе опять будет путаница).

### Модуль A: Reaction Speed
Режимы:
- Simple reaction (нажать при сигнале)
- Go/No‑Go (нажимать только при “разрешённом” сигнале)

Метрики:
- среднее время реакции
- ошибки (false positives / misses)
- точность
- лучший результат

### Модуль B: Working Memory (Sequence)
Простой “повтори последовательность”:
- показать 3–8 элементов
- пользователь повторяет
- сложность растет

Метрики:
- максимальная длина
- точность
- время ответа

### Критерии готовности
- У каждой игры есть Setup → Session → Result → запись в БД → график в /stats
- Игры работают и на Android, и на ноутбуке.
